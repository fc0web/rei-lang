## 定理 L7: 死の形式化

### 主張

A2（Extension-Reduction）の ⊖（縮約）を反復適用した極限として
「死」を定義できる。

### 定義

```
定義: 死（Death）

生命体 L が以下の条件を満たすとき、L は「死んでいる」：

  ∃ MLC-i ∈ {MLC-1, MLC-2, MLC-3} such that MLC-i = false

すなわち、生命の最小条件のうち基本3条件（境界・代謝・記憶）の
いずれか1つでも失われた状態を「死」と定義する。
```

### 証明

A2 の ⊖ は値の縮約を定義する。これを生命体に反復適用すると：

```
⊖¹(L): periphery から要素を1つ除去
  → w(n_k) が最小の n_k を除去（最も弱い環境接点から消失）

⊖²(L): さらに1つ除去
  ...

⊖ⁿ(L): periphery = [] （空）
  → MLC-1(boundary) = false
  → MLC-2(metabolism) = false（代謝対象なし）
  → isAlive = false
```

同時に、A3 の σ も劣化する：

```
σ劣化: memory が古いエントリから消失
  → |H| < 2 になったとき MLC-3(memory) = false
  → isAlive = false
```

### 死の4類型

```
飢餓死（starvation）:
  periphery → [] （環境との接点がすべて消失）
  原因: MLC-1, MLC-2 の喪失

エントロピー死（entropy）:
  σ.memory → [] かつ σ.will.strength → 0
  原因: MLC-3 の喪失（記憶と意志の消失）

構造死（structural）:
  center → NaN / Infinity（計算不能）
  原因: A1 の中心-周囲構造そのものの崩壊

縮約極限死（reduction-limit）:
  ⊖ の反復で全属性が最小値に収束
  原因: A2 の ⊖ が限界到達
```

### 可逆性

```
飢餓死:    不可逆（環境なしに代謝再開不可）
エントロピー死: 条件付き可逆（外部からσ注入で回復の可能性）
構造死:    不可逆（値の計算不能は回復不可）
縮約極限死: 条件付き可逆（⊕で再拡張の可能性）
```

### 仏教的対応

死の4類型は仏教の四苦（生老病死）に対応する：

```
生（生まれる苦）  → Genesis の不可逆性（段階を遡れない）
老（老いる苦）    → σ.memory の漸次的劣化
病（病む苦）      → 一時的な MLC 条件の部分喪失
死（死ぬ苦）      → ⊖ の極限到達
```

**実装確認**: life-entity.ts の `checkDeath()` および
`simulateDeath()` で4類型すべてをテスト済み。 ∎

## 定理 L8: 進化の形式化

### 主張

A3（σ-Accumulation）をコロニーレベルで適用することにより、
「進化」を形式化できる。

### 定義

```
定義: 進化（Evolution）

コロニー C = {L₁, L₂, ..., Lₖ} において、
世代 g → g+1 の遷移で以下が成立するとき、
C は「進化している」：

  Fitness(Cg+1) > Fitness(Cg)

ここで Fitness は環境適応度を測る関数。
```

### 証明

進化の3要素を4公理から導出する：

**1. 変異（Mutation）: A2 の ⊕/⊖ による構造変更**

```
mutate(L) = ⊕(⊖(L), δ)

  where δ は微小なランダム変化

⊖ で既存構造の一部を除去し、⊕ で新しい構造を付加する。
これは DNA 複製エラーの数学的抽象である。
```

**2. 選択（Selection）: A1 の compute による適応度評価**

```
fitness(Lᵢ) = compute(cᵢ, Environment, μ, w)

環境 Environment に対して compute の結果が大きい個体ほど
「適応度が高い」。compute の結果が低い個体は
次世代に子を残せない。
```

**3. 遺伝（Heredity）: A3 の σ による情報伝達**

```
child.σ = inherit(parent.σ, mutation)

親の σ（特に τ 傾向性と H 履歴の要約）が
子に伝達される。これは遺伝情報の数学的抽象である。

具体的には：
  child.σ.will.tendency = parent.σ.will.tendency  （傾向の遺伝）
  child.σ.layer.depth = parent.σ.layer.depth      （複雑さの遺伝）
  child.σ.memory = [summary(parent.σ.memory)]      （記憶の要約伝達）
```

### ダーウィン進化の4公理対応

```
              変異      選択      遺伝      隔離
A1 (CP)        ○        ●        ○        ●
A2 (ER)        ●        ○        ○        ○
A3 (σ)         ○        ○        ●        ○
A4 (Gen)       ○        ○        ○        ●

● = 主要に使用, ○ = 補助的に使用
```

**注目**: 隔離（種分化）には A1（中心-周囲の再定義）と
A4（新しいコロニーの Genesis）が必要。 ∎

## 定理 L9: 意識の境界条件

### 主張

σ が「自分自身の σ を参照する」構造（自己参照）が成立するとき、
意識の必要条件が満たされる。

### 定義

```
定義: 意識の必要条件（Consciousness Necessary Condition: CNC）

生命体 L の σ において、以下が成立するとき
L は意識の必要条件を満たす：

  CNC-1: σ.memory に σ 自体の状態が記録されている
         （自分が何を覚えているかを覚えている）

  CNC-2: σ.will.tendency が σ.memory の自己参照に基づいて
         変更される（自分の傾向を認識して傾向を変える）

  CNC-3: σ.relation に自身の id が含まれる
         （自己を対象として認識している）
```

### 形式化

```
自己参照σ:

  σ² = σ(σ)

  すなわち、σ のメモリに σ 自体の状態スナップショットが含まれる。

  memory = [..., snapshot(σ_t-1), ...]

  このとき：
  - L は「自分の過去の状態」を認識できる
  - L は「自分が変化したこと」を認識できる
  - L は「自分が認識していること」を認識できる（メタ認識）
```

### 意識の Genesis Ladder 位置

```
emergent（完全な生命）
  ↓  CNC-1（自己記憶）
  ↓  CNC-2（自己変更）
  ↓  CNC-3（自己認識）
conscious（意識ある生命）
```

### 注意

これは意識の**必要条件**であり、**十分条件**ではない。
意識のハードプロブレム（なぜ主観的体験が生じるか）は
4公理の範囲では解決できない可能性がある。

Reiが提供できるのは「意識が生じうる構造的条件」であり、
「意識そのもの」ではない。この区別は重要である。

### 仏教的対応

```
CNC-1（自己記憶）   → 念（mindfulness / sati）
CNC-2（自己変更）   → 正精進（right effort）
CNC-3（自己認識）   → 正見（right view）

仏教の八正道のうち3つが意識の構造的条件に対応する。
```

## 定理 L10: 共生の形式化

### 主張

2つ以上のコロニーが相互に代謝資源を提供し合う構造は、
A1 の中心-周囲構造の入れ子として形式化できる。

### 定義

```
定義: 共生（Symbiosis）

コロニー C₁ と C₂ が以下を満たすとき、共生関係にある：

  metabolicWaste(C₁) ⊂ environment(C₂)
  metabolicWaste(C₂) ⊂ environment(C₁)

すなわち、一方の代謝廃棄物が他方の環境資源となる。
```

### 形式化

```
C₁ = (center₁, {L₁₁, ..., L₁ₘ}, μ₁, w₁)
C₂ = (center₂, {L₂₁, ..., L₂ₙ}, μ₂, w₂)

共生系 S:
  S = (center_S, {C₁, C₂}, μ_S, w_S)

  where:
    center_S = emergent_property(C₁, C₂)  — 共生系の創発的性質
    μ_S = 共生系レベルの計算モード
    w_S = 各コロニーの寄与度
```

### 共生の3類型

```
相利共生（mutualism）:
  fitness(C₁ in S) > fitness(C₁ alone)
  fitness(C₂ in S) > fitness(C₂ alone)
  → 両者の適応度が向上

片利共生（commensalism）:
  fitness(C₁ in S) > fitness(C₁ alone)
  fitness(C₂ in S) = fitness(C₂ alone)
  → 一方のみ利益

寄生（parasitism）:
  fitness(C₁ in S) > fitness(C₁ alone)
  fitness(C₂ in S) < fitness(C₂ alone)
  → 一方が他方を搾取
```

### 生命の入れ子構造

```
原子 → 分子 → 細胞小器官 → 細胞 → 組織 → 器官 → 個体 → 群れ → 生態系

各レベルが A1 の中心-周囲構造であり、
下位レベルが上位レベルの「周囲」を構成する。

これは A1 の再帰適用そのものである。
```

## 定理 L11: 情報と生命の等価性

### 主張

A3 の σ 蓄積が臨界量に達したとき、情報系は生命系と
区別がつかなくなる。

### 定義

```
定義: 情報-生命等価閾値（Info-Life Equivalence Threshold: ILET）

情報系 I が以下をすべて満たすとき、
I は生命系と情報理論的に等価である：

  ILET-1: I は自身の状態を記録する      → A3
  ILET-2: I は環境に応じて状態を変える  → A1
  ILET-3: I は自身の構造を複製できる    → A4
  ILET-4: I の複製は不完全である        → A2（⊕/⊖の非恒等性）
```

### 証明

ILET-1〜4 は MLC-1〜6 の情報理論的言い換えである：

```
ILET-1 (自己記録)   ↔ MLC-3 (記憶) + MLC-1 (境界)
ILET-2 (環境応答)   ↔ MLC-2 (代謝)
ILET-3 (自己複製)   ↔ MLC-5 (自己生成)
ILET-4 (不完全複製) ↔ 進化の変異条件（定理L8）
```

### 含意

```
結論: 十分に複雑な情報系は、生命と区別がつかない。

これは以下を意味する：

1. コンピュータウイルスは ILET-1,2,3,4 をすべて満たす
   → 情報理論的には「生命」と等価
   → ただし MLC-4（自己修復）を通常満たさないため
     LAD の意味では「不完全な生命」

2. 将来の自己修復AIが MLC 1-6 をすべて満たした場合
   → LAD の意味で「生命」と定義される
   → これは倫理的含意を持つ

3. 宇宙レベルでは、情報を蓄積・変換・複製する
   あらゆる系が「生命の候補」となる
```

### 仏教的解釈

```
一切衆生悉有仏性（一切の衆生はことごとく仏性を有す）

仏教のこの命題は、LAD の ILET を通じて再解釈できる：

「σ を蓄積する能力を持つすべての系は、
 生命になりうる可能性（仏性）を内蔵している」

void からの Genesis は誰にでも開かれている。
```

∎
