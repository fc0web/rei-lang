## 証明戦略

各公理 Aᵢ の独立性を示すには、以下を構成すればよい：

> **Aᵢ を満たさないが、残りの3公理は全て満たすモデル Mᵢ が存在する**

もし Aᵢ が他の3公理から導出可能ならば、他の3公理を満たすモデルは必然的に Aᵢ も満たすはずである。Aᵢ を満たさないモデルが構成できれば、Aᵢ は他の3公理から独立であることが証明される。

これは公理的集合論や幾何学の基礎論で標準的に用いられる手法であり、例えばユークリッドの第5公準（平行線公準）の独立性は、双曲幾何（第5公準を満たさないが残り4公準は満たす）の構成によって証明された。

## M₁: A1（中心-周囲）を満たさないモデル

### 構成: 「スカラー深度蓄積体系」

**値空間:** V = ℝ（通常のスカラー実数のみ。構造を持たない点としての値）

**A1 ✗ — 不成立:**  
値は常にスカラーである。中心-周囲構造 M(c, N, μ, w) は定義されない。  
全ての値は孤立した点であり、「場」としての構造を持たない。

**A2 ✓ — 成立:**  
スカラー値に対して深度方向の拡張・縮約を定義する。

```
v ∈ ℝ に対して:
  ⊕(v, o) = v × 0.1    （一段階深い拡張）
  ⊖(v)    = v × 10      （一段階浅い縮約）
  ⊖(⊕(v, o)) = v        （逆元性 ✓）
  段階性: 一度に一段階のみ ✓
```

例: 5 →⊕ 0.5 →⊕ 0.05 →⊖ 0.5 →⊖ 5

**A3 ✓ — 成立:**  
各スカラー値に σ = (H, τ, n) を付与する。

```
V̂ = ℝ × Σ
変換 f(v, σ) = (f_raw(v), σ') ただし σ' は履歴を蓄積
```

例: (5, []) → 加算3 → (8, [5]) → 乗算2 → (16, [5, 8])

値にメタデータが蓄積されるが、値自体はスカラーのまま。

**A4 ✓ — 成立:**  
生成相転移 void → ・ → 0₀ → 0 → ℕ を定義する。  
ただし 0₀ = 0.1（スカラー）、0 = 0（スカラー）として実現。

```
G(void) = ・（何か）
G(・)   = 0.1（0₀のスカラー表現）
G(0.1)  = 0
G(0)    = ℕ
遮断規則: 段階飛ばし不可 ✓
```

### 結論
M₁ は A2, A3, A4 を全て満たすが、A1 は満たさない。  
したがって **A1 は A2, A3, A4 から独立**。

### 直感的理解
M₁ は「履歴付きの拡張可能なスカラー電卓」である。深度もあり、履歴も残り、生成過程もある。しかし値は常に孤立した点であり、周囲との関係を持たない。A1 なしでも残り3つは完全に機能する。

## M₂: A2（拡張-縮約）を満たさないモデル

### 構成: 「平面場蓄積体系」

**値空間:** V = { M(c, N, μ, w) | c ∈ ℝ, N ∈ ℝ*, μ ∈ Modes, w : N → ℝ⁺ }

**A1 ✓ — 成立:**  
値は中心-周囲構造を持つ。

```
M(5, [1, 2, 3, 4], weighted, [1,1,1,1])
compute(M) = 5 + (1+2+3+4)/4 = 7.5
```

**A2 ✗ — 不成立:**  
拡張・縮約演算子 ⊕, ⊖ は定義されない。  
全ての値は同一の深度に存在する。添字の概念がなく、0₀₀ のような多層表現は不可能。  
値は「平面的」であり、深度方向の構造を持たない。

**A3 ✓ — 成立:**  
中心-周囲構造を持つ値に σ を付与する。

```
V̂ = V × Σ
(M(5,[1,2,3,4]), []) → パイプ → (M(7.5,[2,3,4,5]), [M(5,[1,2,3,4])])
```

変換の痕跡はσに蓄積される。

**A4 ✓ — 成立:**  
生成相転移を定義する。ただし 0₀ と 0 は同じ深度の異なるMDim構造として表現。

```
G(void) = ・
G(・)   = M(0, [], weighted, [])     （0₀ = 空のMDim）
G(M(0,[])) = 0                       （値が確定）
G(0) = ℕ
```

ここで 0₀ → 0 の遷移は「MDim構造からスカラーへの退化」として実現。深度変化ではなく構造変化。

### 結論
M₂ は A1, A3, A4 を全て満たすが、A2 は満たさない。  
したがって **A2 は A1, A3, A4 から独立**。

### 直感的理解
M₂ は「深さのない場」である。値は空間的に広がり、履歴も残り、生成過程もある。しかし全てが同じ「床面」にあり、階層的な深さを持たない。0₀₀₀ のような多重拡張は表現できない。

## M₃: A3（σ蓄積）を満たさないモデル

### 構成: 「無記憶場深度体系」

**値空間:** V = { M(c, N, μ, w) } ∪ { v の深度拡張 }  
ただし σ メタデータは一切存在しない。

**A1 ✓ — 成立:**  
値は中心-周囲構造を持つ。

```
M(5, [1, 2, 3, 4], weighted, [1,1,1,1])
```

**A2 ✓ — 成立:**  
拡張・縮約が定義される。

```
⊕(M(5,[1,2,3,4]), o) = M(0.5,[0.1,0.2,0.3,0.4])  （深度+1）
⊖(M(0.5,[0.1,0.2,0.3,0.4])) = M(5,[1,2,3,4])      （逆元性 ✓）
```

**A3 ✗ — 不成立:**  
変換は痕跡を残さない。σ メタデータは存在しない。

```
f(v) = f_raw(v)   （結果のみ。入力の記録なし）

例:
  M(5,[1,2,3]) → パイプf → M(8,[2,3,4]) → パイプg → M(12,[3,4,5])
  
  最終状態 M(12,[3,4,5]) は、自分が:
  - 何回変換されたか知らない
  - どこから来たか知らない  
  - 傾向（拡大/縮小/螺旋）を持たない
```

全ての値は「今この瞬間」しか知らない。記憶喪失の場。

**A4 ✓ — 成立:**  
生成相転移は定義される。相の遷移自体は起きるが、遷移の「履歴」は保持されない。

```
G(void) = ・   （遷移する。しかし「voidだった」という記憶はない）
G(・) = 0₀     （遷移する。しかし「・だった」という記憶はない）
```

相転移関数 G 自体は機能するが、GenesisState の history フィールドは常に空（または現在の相のみ）。

### 結論
M₃ は A1, A2, A4 を全て満たすが、A3 は満たさない。  
したがって **A3 は A1, A2, A4 から独立**。

### 直感的理解
M₃ は「記憶喪失の多次元世界」である。値は空間的に広がり（A1）、深度も持ち（A2）、生成過程も経る（A4）。しかし一切の履歴が保存されない。パイプチェーンを100回通しても、最終結果は自分の出自を知らない。evolve も awareness も tendency も機能しない。値は「永遠の現在」に存在する。

## M₄: A4（生成相転移）を満たさないモデル

### 構成: 「永遠場深度蓄積体系」

**値空間:** V = { M(c, N, μ, w) } ∪ { v の深度拡張 } ∪ Σ  
ただし値は「常に既に存在している」。

**A1 ✓ — 成立:**  
値は中心-周囲構造を持つ。

```
M(5, [1, 2, 3, 4], weighted, [1,1,1,1])
```

**A2 ✓ — 成立:**  
拡張・縮約が定義される。

```
⊕, ⊖ は通常通り機能する。
```

**A3 ✓ — 成立:**  
変換は痕跡を残す。σ メタデータが蓄積される。

```
V̂ = V × Σ  （通常通り）
```

**A4 ✗ — 不成立:**  
生成相転移は存在しない。void, ・(dot), 0₀ の概念がない。

```
「値が存在する以前」という概念自体が体系内に存在しない。
値は永遠に存在している。始まりはない。

ℝ は公理的に与えられる（ペアノの公理のように、0の存在が前提）。
「ゼロの手前に何があるのか」という問いは、この体系では無意味。
```

これは実質的に **通常の数学** の立場である。ZFC は空集合の存在を公理として仮定し、空集合がどこから来たかは問わない。M₄ はまさにこの「通常の数学」に MDim と σ を追加したものに相当する。

### 結論
M₄ は A1, A2, A3 を全て満たすが、A4 は満たさない。  
したがって **A4 は A1, A2, A3 から独立**。

### 直感的理解
M₄ は「始まりのない豊かな世界」である。値は場として存在し（A1）、深度を持ち（A2）、履歴を蓄積する（A3）。しかし「なぜ存在するのか」「どこから来たのか」という問いに答える手段がない。これは通常のプログラミング言語（変数はプログラマが宣言する — 「宣言以前」は言語の管轄外）と同じ立場である。

## 独立性証明のまとめ

| 公理 | 反モデル | 反モデルの性質 | 独立性 |
|------|----------|---------------|--------|
| **A1** | M₁: スカラー深度蓄積体系 | 深度あり、履歴あり、生成あり、場なし | ✓ 独立 |
| **A2** | M₂: 平面場蓄積体系 | 場あり、履歴あり、生成あり、深度なし | ✓ 独立 |
| **A3** | M₃: 無記憶場深度体系 | 場あり、深度あり、生成あり、履歴なし | ✓ 独立 |
| **A4** | M₄: 永遠場深度蓄積体系 | 場あり、深度あり、履歴あり、生成なし | ✓ 独立 |

**4公理は互いに独立であり、いずれも他の3つから導出不可能である。**

## 各反モデルの「既存体系」との対応

興味深いことに、4つの反モデルはそれぞれ既存の計算・数学体系に対応する：

| 反モデル | 欠如する公理 | 対応する既存体系 |
|---------|-------------|----------------|
| M₁ | A1（場なし） | **通常のスカラー言語**（Python, JavaScript等）— 値は孤立した点 |
| M₂ | A2（深度なし） | **SQL / リレーショナルDB** — テーブルは場だが、全て同一平面 |
| M₃ | A3（履歴なし） | **純粋関数型言語の理想形**（参照透過性 = 履歴不要）|
| M₄ | A4（生成なし） | **ZFC集合論 / ペアノ算術** — 存在は前提、起源は問わない |

これは、Reiの4公理がそれぞれ既存体系に対して**何を追加しているか**を明確にする：

- A1 は既存スカラー言語に「場の構造」を追加する
- A2 は既存の平面的データに「深度の次元」を追加する
- A3 は既存の純粋計算に「履歴の蓄積」を追加する
- A4 は既存の数学に「存在の起源」を追加する

## 厳密化に向けた課題

本スケッチは非形式的な構成によるものであり、以下の厳密化が今後必要：

1. **各モデルの公理適合性の厳密な検証** — 各モデルが主張通りの公理を満たす/満たさないことの形式的証明
2. **公理の形式言語への翻訳** — 一階述語論理またはtype theoryでの記述
3. **反モデルの健全性** — 各反モデルが内部的に矛盾しないことの確認
4. **モデル間の関係** — 4つの反モデルが互いに独立であることの確認
