## 定理

> **Rei の 4 公理 A1, A2, A3, A4 は互いに独立である。**
> すなわち、いずれの公理も残り 3 つから導出不可能である。

---

## 証明手法

モデル論の標準手法を用いる：

> 公理 Aᵢ が他の 3 公理から導出可能であれば、
> 他の 3 公理を満たす任意のモデルは必然的に Aᵢ も満たす。
> 対偶を取ると：**Aᵢ を満たさないが残り 3 公理を満たすモデル Mᵢ が存在すれば、
> Aᵢ は他の 3 公理から独立である。**

本証明は、4 つの反モデル M1–M4 を TypeScript で構成し、
各モデルが正確に 3 公理を満たし 1 公理を満たさないことを
65 の自動化テストで検証する。

これは Peano 公理や ZFC 公理の独立性証明で用いられる手法と
同一の論理的構造を持つ。

## 公理の形式的要件

### A1: 中心-周囲（Center-Periphery）
1. 値は center と periphery を持つ
2. compute(center, periphery) で値を算出できる
3. periphery が空なら center に退化する
4. periphery の要素数 k ≥ 1 の構造が構成可能

### A2: 拡張-縮約（Extension-Reduction）
1. ⊕ (extend) が定義される
2. ⊖ (reduce) が定義される
3. ⊖(⊕(v, s)) = v （逆元性）
4. 一段階ずつの操作（段階性）
5. 深度 d ≥ 2 の値が構成可能

### A3: σ蓄積（Sigma Accumulation）
1. 変換は履歴 H に記録される
2. 変換回数 n が追跡される
3. 傾向性 τ が計算可能
4. 履歴は消去不可能（蓄積のみ）

### A4: 生成相転移（Genesis Phase Transition）
1. void 状態が存在する
2. 相転移関数 G が定義される
3. 遮断規則: 段階飛ばし不可
4. 最低 4 段階の相が存在する

## 反モデル M1: ScalarModel — A1 の独立性

**構成:** スカラー深度蓄積体系

**値空間:** V = { (s, d, σ) | s ∈ ℝ, d ∈ ℕ, σ ∈ Σ }

値は常にスカラー（実数）。中心-周囲の「場」構造を持たない。

| 公理 | 充足 | 根拠 |
|------|------|------|
| **A1** | **✗** | periphery が存在しない。compute(center, periphery) は定義不能。値は孤立した点。 |
| A2 | ✓ | ⊕(v) = v×0.1（depth+1）、⊖(v) = v×10（depth−1）。逆元性 ⊖(⊕(v))=v は数値的に検証済み。 |
| A3 | ✓ | 各変換で (value, sigma) → (newValue, sigma') として履歴・回数・傾向性を蓄積。 |
| A4 | ✓ | void → dot → zero-ext → zero → natural の5相を段階的に遷移。遮断規則あり。 |

**直感的理解:** M1 は「履歴付きの拡張可能なスカラー電卓」。Python や JavaScript のスカラー変数に相当する。

**テスト:** §1 (tests 1.1–1.16) — 16 テストで検証

**∴ A1 は A2, A3, A4 から独立である。** ∎

## 反モデル M2: FlatModel — A2 の独立性

**構成:** 平面場蓄積体系

**値空間:** V = { (c, N, σ) | c ∈ ℝ, N ∈ ℝ*, σ ∈ Σ }

値は中心-周囲構造を持つが、深度の次元が存在しない。全ての値が同一平面にある。

| 公理 | 充足 | 根拠 |
|------|------|------|
| A1 | ✓ | center, periphery あり。compute = center + mean(periphery)。退化条件あり。 |
| **A2** | **✗** | extend/reduce は定義されない。depth は常に 0。添字空間が存在しない。0₀₀₀ のような多層表現は不可能。 |
| A3 | ✓ | 変換で (value, sigma) を蓄積。履歴・回数・傾向性あり。 |
| A4 | ✓ | void → dot → zero-field → zero → natural。0₀ は空の MDim として表現。 |

**直感的理解:** M2 は「深さのないスプレッドシート」。SQL のリレーショナルテーブルに相当する。

**テスト:** §2 (tests 2.1–2.14) — 14 テストで検証

**∴ A2 は A1, A3, A4 から独立である。** ∎

## 反モデル M3: AmnesicModel — A3 の独立性

**構成:** 無記憶場深度体系

**値空間:** V = { (c, N, d) | c ∈ ℝ, N ∈ ℝ*, d ∈ ℕ }（σ なし）

値は場であり深度も持つが、変換の履歴を一切保持しない。

| 公理 | 充足 | 根拠 |
|------|------|------|
| A1 | ✓ | center, periphery, compute あり。 |
| A2 | ✓ | extend/reduce あり。逆元性あり。多段階拡張可能。 |
| **A3** | **✗** | σ が存在しない。getHistory() は常に空。getTransformCount() は常に 0。3回連鎖変換しても来歴不明。 |
| A4 | ✓ | void → dot → zero-ext → zero → natural。ただし遷移の履歴は保持されない。 |

**直感的理解:** M3 は「記憶喪失の多次元世界」。純粋関数型言語（参照透過性 = 履歴不要）に相当する。

**テスト:** §3 (tests 3.1–3.15) — 15 テストで検証

**重要な観察:** M3 では `m3Chain(v, [f, g, h])` を実行すると結果の値は正しいが、
「f を通った」「g を通った」「元の値は何だったか」という情報は一切残らない（test 3.10）。
これは A3 の不在が**undo, evolve, awareness, tendency** を全て不可能にすることを示す。

**∴ A3 は A1, A2, A4 から独立である。** ∎

## 反モデル M4: EternalModel — A4 の独立性

**構成:** 永遠場深度蓄積体系

**値空間:** V = { (c, N, d, σ) | c ∈ ℝ, N ∈ ℝ*, d ∈ ℕ, σ ∈ Σ }

値は場であり、深度もあり、履歴も蓄積される。しかし「存在の起源」がない。

| 公理 | 充足 | 根拠 |
|------|------|------|
| A1 | ✓ | center, periphery, compute あり。 |
| A2 | ✓ | extend/reduce あり。逆元性あり。 |
| A3 | ✓ | σ 蓄積あり。履歴・回数・傾向性あり。 |
| **A4** | **✗** | genesisCreate() は null を返す。genesisForward() は例外を投げる。void の概念がない。値は公理的に「すでに存在している」。 |

**直感的理解:** M4 は「始まりのない豊かな世界」。ZFC 集合論やペアノ算術に相当する。
「0 は自然数である」と宣言し、「0 はどこから来たのか」は問わない。

**テスト:** §4 (tests 4.1–4.15) — 15 テストで検証

**重要な観察:** M4 の `m4Axiom()` は値 0 を返すが、これは Genesis で生成されたものではなく、
ペアノの公理と同様に**前提として与えられる**（test 4.13）。
「値が存在する以前」という問いは M4 の体系内では**無意味**である（test 4.14）。

**∴ A4 は A1, A2, A3 から独立である。** ∎

## 独立性行列

```
        A1    A2    A3    A4
M1:     ✗     ✓     ✓     ✓     → A1 は独立
M2:     ✓     ✗     ✓     ✓     → A2 は独立
M3:     ✓     ✓     ✗     ✓     → A3 は独立
M4:     ✓     ✓     ✓     ✗     → A4 は独立
```

対角線上に正確に 4 つの ✗ が並ぶ。各行に正確に 1 つの ✗、各列に正確に 1 つの ✗。

## 反モデルと既存体系の対応

| 反モデル | 欠如する公理 | 対応する既存体系 |
|---------|-------------|----------------|
| M1 | A1（場なし） | **Python / JavaScript** — 値は孤立した点 |
| M2 | A2（深度なし） | **SQL / リレーショナルDB** — テーブルは場だが同一平面 |
| M3 | A3（履歴なし） | **純粋関数型言語** — 参照透過性 = 履歴不要 |
| M4 | A4（生成なし） | **ZFC集合論 / ペアノ算術** — 存在は前提、起源は問わない |

これは Rei の 4 公理が既存体系に対して**何を追加しているか**を明確にする：

- **A1** は「値を点から場に拡張する」
- **A2** は「値に深度の次元を与える」
- **A3** は「計算に時間の痕跡を刻む」
- **A4** は「存在の起源を問う」

## 証明の形式的強度について

本証明は、以下の意味で**計算論的独立性証明（computational independence proof）**である：

1. **構成的:** 4 つの反モデルを具体的に構成している（存在証明ではなく構成証明）
2. **検証可能:** 65 の自動化テストが各モデルの公理適合性を検証する
3. **再現可能:** `npx vitest run tests/axiom-independence.test.ts` で誰でも再現できる

一方で、以下の点は本証明の範囲外であり、今後の厳密化課題として残る：

- **公理の一階述語論理への完全な翻訳** — 現在は半形式的（型定義 + テスト）
- **反モデルの無矛盾性の独立証明** — 各モデルがTypeScriptの型システムを満たすことで暗黙に保証されているが、明示的な証明はなし
- **公理の完全性（completeness）** — 4 公理が Rei の全ての定理を導出するのに十分であるかの証明は別の問題
