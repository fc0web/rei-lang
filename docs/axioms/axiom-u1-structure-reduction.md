# 公理U1：構造還元公理（Structure Reduction Axiom）— Rei構文設計

## 公理の定義

**公理U1：構造還元公理（Structure Reduction Axiom）**

*任意の知識領域 D における基本単位は、有限回の操作で中心-周囲パターン 𝕄{c; p₁, p₂, ...} に還元できる。この還元を「構造射影（structural projection）」と呼ぶ。*

```
∀ D（知識領域）, ∀ s ∈ D（基本単位）:
  ∃ π : s → 𝕄{c; p₁, ..., pₙ}
  
  ここで π は構造射影であり：
    c = s の注目点（focus）
    pᵢ = s の文脈要素（context elements）
```

**この公理が主張すること：**
物理学の粒子も、社会学の個人も、言語学の語も、数学の点も、
全ては「何かを中心に据え、その周囲に文脈が広がる」構造に分解できる。
この分解が可能であるがゆえに、Reiの 𝕄 は普遍的な記述単位となる。

## 哲学的根拠

U1は二つの哲学的伝統の合流点に位置する。

**西洋：現象学の「志向性（Intentionality）」**

フッサールの現象学において、意識は常に「何かについての意識」である。
知覚には必ず「ノエシス（意識の作用 = 注目）」と「ノエマ（意識の対象 = 文脈）」がある。
つまり、認識の最小単位は「中心（注目するもの）と周囲（注目されるもの）」の対である。

𝕄{c; p₁, ..., pₙ} はこの構造の形式化である。

**東洋：仏教の「五蘊（pañcaskandha）」**

仏教では、あらゆる存在は五蘊（色・受・想・行・識）に分解できるとする。
そしてどの蘊も「それ自体で独立に存在するのではなく、他の蘊との関係の中にある」。

これは「全てが中心-周囲のパターンに還元できる」というU1と正確に対応する。
五蘊のどれか一つを中心に据えれば、残りが周囲になる。
どれを中心にするかによって、同じ存在の異なる側面が見える。

**D-FUMTの統一関数との接続：**

```
U(x,t) = α·f(x) + β·g(t) + γ·h(x,t)
```

統一関数の3項構造も、実はU1の一例である：
- f(x) を中心に見れば → 𝕄{f(x); g(t), h(x,t)}（空間中心の見方）
- g(t) を中心に見れば → 𝕄{g(t); f(x), h(x,t)}（時間中心の見方）
- h(x,t) を中心に見れば → 𝕄{h(x,t); f(x), g(t)}（相互作用中心の見方）

同じ対象を異なる中心で射影できる — これがU1の力である。

## 構造射影 π の具体例

### 例1：物理学 — ニュートン力学

```
対象：質点系の運動

従来の記述：
  F = ma
  F₁₂ = -G·m₁·m₂/r² · r̂

Reiへの構造射影 π：
  𝕄{particle_i; particle_1, particle_2, ..., particle_n}
  
  中心 = 注目する質点
  周囲 = 他の全質点（力の源）
  σ.field = 重力場（周囲からの力の合成）
  σ.flow = 運動量ベクトル
  σ.memory = 軌道の来歴
```

### 例2：言語学 — 文の構造

```
対象：自然言語の文 "The cat sat on the mat"

Reiへの構造射影 π：
  𝕄{"sat"; "cat", "on", "mat"}
  
  中心 = 動詞（文の意味的焦点）
  周囲 = 項（動作の参加者と場所）
  σ.field = 統語的関係（主語・目的語・場所）
  σ.flow = 時制（過去 → 現在への方向）
  σ.memory = 文脈（この文の前に何が語られたか）
```

### 例3：生物学 — 細胞

```
対象：細胞の構造と機能

Reiへの構造射影 π：
  𝕄{nucleus; mitochondria, ribosome, ER, membrane}
  
  中心 = 核（DNAを持つ情報の源）
  周囲 = 小器官（核の指示を実行する）
  σ.field = 細胞内シグナル場
  σ.flow = 代謝の流れ
  σ.memory = DNA（来歴 = 進化の記録）
  σ.tendency = 分化方向（:expand=分裂, :rest=G0期）
```

### 例4：経済学 — 市場

```
対象：市場均衡

Reiへの構造射影 π：
  𝕄{price; buyer_1, buyer_2, ..., seller_1, seller_2, ...}
  
  中心 = 価格（市場の注目点）
  周囲 = 買い手と売り手（価格を動かす力）
  σ.field = 需給バランス
  σ.flow = 価格変動の方向
  σ.tendency = :rest（均衡志向）or :oscillate（バブル/暴落サイクル）
```

### 例5：音楽 — 和声

```
対象：Cメジャーコード

Reiへの構造射影 π：
  𝕄{C; E, G}
  
  中心 = 根音（コードの基盤）
  周囲 = 構成音（根音との音程関係）
  σ.field = 音響的近傍（倍音列）

  σ.tendency = :contract（トニック=安定）or :expand（ドミナント=解決要求）
```

## 射影の一意性について

**重要：構造射影 π は一意ではない。**

```
同じ対象 s に対して、複数の射影が存在し得る：

  π₁(文) = 𝕄{"sat"; "cat", "on", "mat"}     ← 動詞中心
  π₂(文) = 𝕄{"cat"; "sat", "on the mat"}     ← 主語中心
  π₃(文) = 𝕄{"mat"; "cat sat on"}            ← 場所中心

これは欠陥ではなく、設計の意図である。
「何を中心に見るか」によって同じ対象の異なる側面が見える。
```

これは仏教の **空（śūnyatā）** の構造に対応する。
対象には「固有の本質」がなく、**見方（射影）によって現れ方が変わる**。

**射影の集合としての完全記述：**

```
complete(s) = { π₁(s), π₂(s), ..., πₖ(s) }

全ての可能な射影の集合が、対象 s の「完全な記述」を構成する。
単一の射影は対象の一面しか捉えない。
```

## Rei構文への落とし込み

### 構造射影の明示的記述

```rei
# ── 構造射影の宣言 ──

射影 mechanics(particle, system) =
  𝕄{particle; system |> filter(≠ particle)...}

# 使用
let solar = [sun, mercury, venus, earth, mars, jupiter, saturn, uranus, neptune]
let earth_view = mechanics(earth, solar)
# → 𝕄{earth; sun, mercury, venus, mars, jupiter, saturn, uranus, neptune}

earth_view |> sigma.field       # → 重力場（他の天体からの力）
earth_view |> compute :weighted  # → 地球への合成力
```

### 射影の切り替え

```rei
# ── 同じ系を異なる中心で見る ──

let sentence = ["cat", "sat", "on", "mat"]

let verb_view = 𝕄{"sat"; "cat", "on", "mat"}
let subj_view = 𝕄{"cat"; "sat", "on", "mat"}

# ★ 同じデータだが、異なる射影 → 異なる計算結果
verb_view |> compute :weighted   # → 動作の意味的重心
subj_view |> compute :weighted   # → 主体の意味的重心
```

### 射影関数 `project`

```rei
# ── 汎用的な射影パイプコマンド ──

let data = [a, b, c, d, e]

data |> project(center: a)    # → 𝕄{a; b, c, d, e}
data |> project(center: c)    # → 𝕄{c; a, b, d, e}

# 条件による中心の自動選択
data |> project(center: :max)  # → 最大値を中心に据える
data |> project(center: :min)  # → 最小値を中心に据える
```

### 領域定義構文

```rei
# ── 新しい知識領域をReiに教える ──

領域 chemistry {
  構造射影 = molecule |> project(center: :most_connected)
  
  場の解釈 = σ.field → "chemical bonds"
  流れの解釈 = σ.flow → "reaction direction" 
  記憶の解釈 = σ.memory → "synthesis history"
}

let water = molecule("H2O")
water |> chemistry.project
# → 𝕄{O; H, H}（酸素が中心、水素が周囲）
```

## 数学的定理

```
定理 U1.1（射影の存在定理）：
  任意の有限構造 s に対して、少なくとも一つの構造射影 π が存在する。
  
  証明の概略：s の要素から任意の一つを選んで center とし、
  残りを periphery とすれば 𝕄 が構成できる。
  要素が有限である限り、この操作は常に可能。■

定理 U1.2（射影の多重性定理）：
  n 要素の構造 s に対して、少なくとも n 個の異なる射影が存在する。
  
  証明：各要素を中心として選ぶことで n 通りの射影が得られる。■

定理 U1.3（射影の情報保存性）：
  構造射影 π は情報を失わない。すなわち、
  𝕄{c; p₁, ..., pₙ} から元の構造 s を復元できる。
  
  π⁻¹(𝕄{c; p₁, ..., pₙ}) = s

定理 U1.4（C1との整合性）：
  構造射影後の 𝕄 に対して、sigma は完全に機能する。
  すなわち、U1 は C1〜C5 の上位互換ではなく、相補的である。
  
  U1 = 外部構造を 𝕄 に変換する
  C1〜C5 = 𝕄 になった値の内部構造を扱う
```
