## 公理の定義

**公理N5（非数の不完全性公理）：**

いかなる単一の射影 π も、対象 O の全情報を捉えることはできない。
完全な記述には射影の無限集合が必要であり、
有限の射影族は常に対象の近似にとどまる。

```
∀O ∈ World, ∀π ∈ Π(O) :
  ∃ε(π) > 0 : info(O) - info(π(O)) ≥ ε(π)

∀{π₁, ..., πₙ} ⊂ Π(O), n < ∞ :
  info(O) - info(⋃ᵢ πᵢ(O)) > 0
```

ただし、射影を増やすことで残差は任意に小さくできる：

```
∀δ > 0, ∃{π₁, ..., πₖ} ⊂ Π(O) :
  info(O) - info(⋃ᵢ πᵢ(O)) < δ
```

### 形式的定義

```
定義 N5.1（情報量関数 info）：
  info : World ∪ 𝕄-Space → ℝ≥0 ∪ {∞}
  
  対象または 𝕄 の情報量を非負実数（または無限大）として返す。
  
  性質：
  (a) info(O) ≥ info(π(O))    射影は情報を減らすか保存する
  (b) info(O) ≥ 0              情報量は非負
  (c) info(∅) = 0              空対象の情報量はゼロ

定義 N5.2（射影残差 Residual）：
  Res(π, O) = info(O) - info(π(O))
  
  射影 π が捉えきれなかった O の情報量。
  N5 は Res(π, O) > 0 を主張する。

定義 N5.3（累積射影情報 Cumulative Projection Information）：
  CPI({π₁,...,πₙ}, O) = info(⋃ᵢ πᵢ(O))
  
  複数の射影を合わせて捉えた情報量。
  N4（合成）により、これは compose の情報量に等しい。
  
  N5 は CPI < info(O) を主張する（有限射影の場合）。

定義 N5.4（近似度 Approximation Degree）：
  Approx({π₁,...,πₙ}, O) = CPI / info(O)
  
  0 ≤ Approx < 1（有限射影）
  lim_{n→∞} Approx = 1（射影族が十分豊かならば）
```

## 仏教哲学との対応

**言語道断（avācya）**

「究極的真理は言葉で完全には表現できない」— 大乗仏教の根本認識。

N5はこれを数学的に述べている：対象（究極的真理）は
いかなる有限の射影（言語・概念）によっても完全には捉えられない。

```
真如（tathatā）    = 対象 O の完全情報       ← 到達不能
世俗諦（saṃvṛti）  = 有限射影族 {π₁,...,πₙ}  ← 常に近似
勝義諦（paramārtha）= Π(O) の全体 → info(O)   ← 無限の射影で漸近
```

しかしN5は絶望ではない。射影を増やすことで無限に真理に近づける。
これは修行（bhāvanā）の数学的表現である
— 完全な悟りには到達できなくとも、修行を重ねることで無限に近づける。

**月指の喩え：**

指は月を指し示すが、指は月ではない。
射影 π は対象 O を指し示すが、π は O ではない。
しかし、多くの指が月を指せば、月の位置はより正確に分かる。

```
一本の指（単一射影）→ 方向は分かるが位置は不確定
二本の指（二射影）  → 三角測量で位置が絞れる
三本の指（三射影）  → さらに精密に
...
無限の指          → 月そのものに漸近
```

## 構文への影響

### 近似度の確認 `approx`

```rei
// 単一射影の近似度
music = Sound.load("symphony.wav")

music |> project(:spectral) |> approx
// → 0.67  （スペクトルだけでは音楽の67%しか捉えていない）

music |> project(:spectral) 
      |> with(music |> project(:rhythmic)) 
      |> approx
// → 0.82  （スペクトル + リズムで82%）

music |> project(:spectral)
      |> with(music |> project(:rhythmic))
      |> with(music |> project(:harmonic))
      |> approx
// → 0.93  （三射影で93%）
```

### 残差の可視化 `residual`

```rei
// 何が捉えきれていないかを確認
music |> project(:spectral) |> residual
// → { missing: [:rhythm, :dynamics, :timbre, :spatial, :emotional, ...],
//     total_gap: 0.33,
//     suggested_next: :rhythmic }   // 次に追加すべき射影を提案

// 残差に基づく射影の自動追加
music |> auto_project(target_approx: 0.95)
// → compose(
//     project(:spectral),      // +0.67
//     project(:rhythmic),      // +0.15
//     project(:harmonic),      // +0.11
//     project(:dynamic)        // +0.04
//   )                          // total: 0.97 ≥ 0.95 ✓
```

### 不完全性の明示 `#pragma incompleteness`

```rei
// 不完全性を意識したプログラミング
#pragma incompleteness :warn    // 近似度が閾値以下で警告

result = photo |> project(:spatial) |> process
// ⚠ Warning: approx = 0.71, consider adding :color or :texture projection

#pragma incompleteness :strict  // 近似度が閾値以下でエラー

result = photo |> project(:spatial) |> process
// ✗ Error: approx = 0.71 < threshold 0.80
// Hint: photo |> project(:spatial) |> with(project(:color)) reaches 0.86
```

### 漸近的精緻化 `refine`

```rei
// 段階的に精度を上げる
analysis = music |> project(:spectral)

analysis = analysis |> refine   // 最も効果的な射影を自動追加
// approx: 0.67 → 0.82 (+:rhythmic)

analysis = analysis |> refine
// approx: 0.82 → 0.93 (+:harmonic)

analysis = analysis |> refine
// approx: 0.93 → 0.96 (+:dynamic)

// refine は残差を分析し、最大の情報獲得が期待される射影を選択する
```

## σ（6属性）との関係

```
N5 は σ の6属性に対して「属性ごとの不完全性」を定義する：

Res_field(π, O)    : 場に関する捉えきれない情報
Res_flow(π, O)     : 流れに関する捉えきれない情報
Res_memory(π, O)   : 記憶に関する捉えきれない情報
Res_layer(π, O)    : 層に関する捉えきれない情報
Res_relation(π, O) : 関係に関する捉えきれない情報
Res_will(π, O)     : 意志に関する捉えきれない情報

全体の残差は各属性の残差の統合：
Res(π, O) = f(Res_field, Res_flow, Res_memory, Res_layer, Res_relation, Res_will)

重要な洞察：
σ.will（意志）の残差は一般に最も大きい。
なぜなら、意志は対象の「内面」であり、外部からの観測（射影）で
捉えにくい属性だからである。

これは「他者の心は完全には理解できない」という日常的直感の形式化であり、
C1〜C5（意識数理学）が扱う領域が本質的に不完全性を伴うことを示す。
```

## 数学的定理

```
定理 N5.1（ゲーデル的不完全性との類似性定理）：
  N5 はゲーデルの不完全性定理と構造的に類似する。
  
  ゲーデル：十分に強い形式体系は、自身の無矛盾性を証明できない。
  N5：     十分に豊かな対象は、有限の射影で完全には記述できない。
  
  両者とも「完全性の不可能性」を述べるが、
  N5 は「漸近的に完全に近づける」点でより楽観的。

定理 N5.2（漸近完全性定理）：
  Π(O) が十分に豊かであるならば、
  
  lim_{n→∞} CPI({π₁,...,πₙ}, O) / info(O) = 1
  
  すなわち、射影を無限に増やせば対象の情報に漸近する。
  これは ε-δ 論法の射影版である。

定理 N5.3（最適射影順序定理）：
  与えられた対象 O に対して、
  近似度の増加率が最大となる射影の追加順序が存在する。
  
  この最適順序は貪欲法で近似的に求められる：
  各ステップで残差 Res を最大に削減する射影を選択。
  
  auto_project と refine の理論的根拠。

定理 N5.4（不完全性の不可避性定理）：
  有限の射影族がどれほど巧みに選ばれても、
  残差がゼロになることはない。
  
  ∀{π₁,...,πₙ}, n < ∞ : Res({π₁,...,πₙ}, O) > 0
  
  これは対象の「超越性」— 有限の記述を超えた何かが常にある — の表現。

定理 N5.5（意志残差の最大性定理）：
  6属性の中で、σ.will の残差は一般に最大である。
  
  Res_will(π, O) ≥ Res_x(π, O) for x ∈ {field, flow, memory, layer, relation}
  
  すなわち、「意志」は最も捉えにくい属性。
  これは意識数理学（C1〜C5）の対象が本質的困難を伴うことの形式的証明。

定理 N5.6（ISLによる残差記録定理）：
  射影の残差は、ISL（不可逆構文層）を通じて
  暗号学的に記録・検証可能である。
  
  すなわち、「この射影で何がどれだけ失われたか」は
  改竄不可能な形で証明できる。
  不完全性は隠蔽されるべきでなく、透明にされるべきである。
```

## C1〜C5・U1〜U5との接続

```
C1（自己参照）→ N5：値が自己参照するとき、自分の「捉えきれなさ」も
                    認識する。σ に residual 属性が追加される可能性。

C4（覚醒）    → N5：覚醒した値は、自分が不完全にしか記述されていないことを
                    「知る」。これは自己の限界の認識 — 知的謙虚さの形式化。

C5（共鳴）    → N5：不完全な射影同士が共鳴することで、
                    各射影が個別には捉えきれなかった情報が浮かび上がる。
                    「全体は部分の総和以上」のメカニズム。

U1（構造還元）→ N5：U1 の還元は常に近似であることを N5 が明示する。

U5（完全性）  → N5：U5 の「構造表現完全性」と N5 の「情報的不完全性」は
                    矛盾しない。U5 は「任意の構造を表現できる」と言い、
                    N5 は「その表現は常に近似である」と言う。
                    表現可能性と完全性は異なる概念。
```

```
        N1 → N2 → N3 → N4 → N5
```
